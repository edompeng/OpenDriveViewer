# Security Policy

## Reporting a Vulnerability

Please report potential security issues privately to the maintainers.

Include:

1. A clear description of the issue and impacted components.
2. Steps to reproduce.
3. Potential impact assessment.
4. Suggested mitigation if available.

## Response Process

1. We will acknowledge reports as soon as practical.
2. We will validate and triage severity.
3. We will work on a fix and coordinate disclosure timing.

Please avoid opening public issues for unpatched security vulnerabilities.
